FFMPEG Executable Path
The path to the ffmpeg executable. (e.g: /usr/bin/ffmpeg or C:\ffmpeg\bin\ffmpeg.exe) FFMPEG version 4.2+ required. Check by opening the version tab.

The ffmpeg path setting is currently locked and can't be edited from the UI. It's not usually necessary to update this path once it's known to be working. Run dizquetv with the --unlock command line argument to enable editing it again.
Stream Mode
Pipeline for IPTV / M3U channel URLs (Tunarr-style modes). Plex Live TV / HDHomeRun always uses classic MPEG-TS (/video) regardless of this setting — required for a continuous live TS pipe. Changing this does not alter HDHR; re-download your M3U after changing modes for IPTV apps.
  • MPEG-TS — Classic dizqueTV: per-program transcode + FFmpeg concat (also what HDHR uses).
  • HLS — M3U points at an HLS playlist of program segments. For IPTV clients that prefer HLS.
  • HLS alt — HLS with stronger normalization between items (more CPU).
  • HLS Direct — Single-item remux playlist (minimal processing). Not for Plex Live TV.
  • HLS Direct v2 — Multi-item HLS playlist preferring remux/copy. Not for Plex Live TV.
Remux container for Direct modes (MPEG-TS is safest for most clients; MKV/MP4 can improve soft subtitle options on compatible players).

Encoder speed vs quality when video is transcoded. Slower = better quality / more CPU. Faster = lower CPU / slightly lower quality. Mapped to FFmpeg -preset for software (libx264/libx265), NVENC, and QSV where supported. Has no effect when video is stream-copied.
FFmpeg -hwaccel for decoding. Use CUDA with h264_nvenc / hevc_nvenc (big help on 4K HEVC). Frames are copied to system memory so scale and subtitle burn still work. When a program has multiple audio or subtitle tracks, dizqueTV / FFmpeg will prefer tracks matching this language for both audio and subtitles (ISO 639 code, e.g. eng, spa). Falls back to the default/selected track if no match is found.

Uses the preferred language above for track selection. Live TV is MPEG-TS — that limits what Plex can toggle.
  • Off — no subtitles from dizqueTV.
  • Soft only — soft-map image tracks (PGS / VOBSUB / DVB) when present so some Plex clients can show a Subtitles menu entry and turn them on/off. Text (SRT/ASS) is not burned and will not appear.
  • Soft + burn text — soft-map image tracks; extract and burn text tracks into the picture (always on; not in the Subtitles menu). First play of a title may pause briefly while text is extracted (cache/subtitles thereafter).
Soft text (SRT/ASS) cannot be carried reliably in MPEG-TS for Plex Live TV selection — that is a container/client limit, not a missing checkbox.
